# Entropy Brief

Use this brief explanation when a repository is blocked from implementation readiness.

AI coding increases entropy because:
* the model sees only part of the system at a time
* it optimizes locally and can miss cross-file or cross-subsystem constraints
* small inconsistencies accumulate across edits
* later changes build on already-drifted assumptions
* without explicit contracts and verification, the system degrades faster than humans notice

Keep the explanation short. The goal is to justify the readiness standard, not to give a long theory lecture.
